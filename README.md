# ARIS – AI Retention Intelligence System

An intelligent customer retention platform that combines **Machine Learning**, **Retrieval-Augmented Generation (RAG)**, and **Agentic AI** to predict customer churn and automatically generate personalised retention strategies for telecom businesses.

## 🌐 Live Demo

**Deployed App:** [https://arischurnai.streamlit.app/](https://arischurnai.streamlit.app/)

## 📖 Overview

ARIS helps subscription-based businesses tackle customer churn by:

1. **Predicting Churn Risk** — An XGBoost classifier estimates churn probability for each customer
2. **Explaining Predictions** — Feature importance scores translated into plain-English risk summaries
3. **Retrieving Market Context** — RAG pipeline fetches relevant segment statistics and domain knowledge
4. **Generating Retention Strategies** — LLM creates personalised, structured retention plans
5. **Enabling Interactive Dialogue** — Agentic chat interface for follow-up questions and deeper exploration

### Technology Stack

- **Streamlit** — Interactive web UI and deployment
- **LangChain** — Agentic workflow orchestration and RAG pipeline
- **Groq (LLaMA 3)** — Fast LLM inference via Groq LPU hardware
- **XGBoost** — Core churn prediction model
- **SMOTE** — Synthetic minority oversampling for class balancing
- **DuckDuckGo Search API** — Live web search for current competitor and market data
- **scikit-learn** — Preprocessing and model utilities
- **imbalanced-learn** — SMOTE implementation

## 🚀 Getting Started

### Prerequisites

- Python 3.11 or higher
- GROQ API key ([Get one here](https://console.groq.com))

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/ARIS.git
   cd ARIS
   ```

2. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Train the ML model**
   ```bash
   python train.py
   ```
   This trains the XGBoost classifier on the Telco dataset and saves the model artifacts.

### Running the Application

```bash
streamlit run app.py
```

Then open your browser to `http://localhost:8501` and enter your GROQ API key in the sidebar.

## ✨ Features

### 1. Customer Churn Predictor (Page 1)
Enter customer details across four sections:
- **Account Details** — Tenure, contract type, payment method, paperless billing
- **Billing** — Monthly charges, total charges
- **Demographics** — Senior citizen status, partner, dependents
- **Services** — Internet type, phone lines, streaming, online security, tech support, and more

The prediction dashboard displays:
- **Churn Probability Gauge** — Visual semi-circular gauge with raw churn probability
- **Risk Tier Badge** — Colour-coded triage (🟢 Low <30% / 🟡 Medium 30–60% / 🔴 High >60%)
- **Prediction Verdict** — *Retention Likely* or *Churn Risk*
- **Top Churn Driver Chart** — Horizontal bar chart of the top 8 feature importances
- **Executive Risk Summary** — 2–3 sentence plain-English explanation auto-generated by the LLM

### 2. AI Retention Agent (Page 2)
An interactive chat interface where retention specialists can:
- **Generate a Full Retention Strategy** — One-click structured plan covering risk assessment, root cause analysis, recommended interventions, personalised messaging, and escalation flags
- **Ask Follow-up Questions** — Natural language queries like *"Why is this customer at risk?"* or *"What if they switched to a one-year contract?"*
- **Receive Clarifying Questions** — For high-risk customers (>50%), the agent proactively asks targeted questions to gather context before finalising recommendations
- **Live Web Search** — Agent invokes DuckDuckGo when internal knowledge is insufficient (e.g., current competitor promotions)
- **Source Attribution** — Retrieved segment statistics shown in a collapsible panel for full transparency

### 3. Batch Analysis (Page 3)
Upload a CSV of multiple customer records and download a results table with churn probability scores for the entire cohort — ideal for identifying the highest-risk segments for proactive outreach campaigns.

## 🤖 Machine Learning Model

### Training Process (`train.py`)

1. Loads the Kaggle Telco Customer Churn dataset (7,043 records; 7,032 after cleaning)
2. Handles missing values and encodes categorical features
3. Applies SMOTE to balance the minority churn class
4. Scales features using StandardScaler
5. Trains an XGBoost classifier with probability threshold tuning
6. Saves model artifacts: `xgboost_model.pkl`, `scaler.pkl`

### Model Performance

| Metric | Value |
|--------|-------|
| Accuracy | 75.37% |
| ROC-AUC Score | 0.836 |
| Recall (Churn class) | 0.76 |
| Precision (Churn class) | 0.52 |
| F1-Score (Churn class) | 0.62 |
| Optimal Threshold (τ*) | 0.52 |

> The recall of **0.76** is the most operationally significant metric — correctly identifying 76% of customers who will actually churn, enabling timely intervention before revenue is lost.

## 🔍 RAG System

### Knowledge Base

The RAG knowledge base comprises two complementary source types:

**Source 1: Historical Churn Segment Index**
Customers from the Telco dataset are grouped by key discriminative attributes — contract type, internet service, tenure band, monthly charge band, and senior citizen status. At inference time, the queried customer's attributes are matched to retrieve aggregated segment statistics: churn rate, median tenure at churn, common exit reasons, and historically effective retention interventions.

**Source 2: Domain Knowledge Dictionary**
A curated, hand-authored knowledge base covering churn driver taxonomy, a retention intervention library (discount schedules, contract upgrade incentives, loyalty credits, win-back offers), segment-specific best practices, and internal business rules (maximum discount percentages, escalation thresholds).

### Retrieval Process

The retrieval function executes a two-stage lookup:
1. **Segment Matching** — Customer's categorical attributes matched against indexed segments; multiple matching segments ranked by statistical relevance
2. **Dictionary Lookup** — Relevant entries retrieved based on keyword overlap between the customer's risk profile and the knowledge dictionary's structured tags

## 🧠 Agentic Retention Agent

The agent extends single-pass RAG with multi-step reasoning:

- **Tool 1: RAG Knowledge Retrieval** — Can re-invoke retrieval mid-conversation as new context emerges
- **Tool 2: DuckDuckGo Live Web Search** — Retrieves current competitor promotions, industry benchmarks, and latest retention best practices (invoked only when internal knowledge is insufficient)
- **Capability 3: Full Retention Strategy Generation** — Structured plan with risk summary, root cause analysis, ranked interventions, personalised outreach scripts, and escalation flags
- **Capability 4: Follow-up Question Generation** — Proactively gathers missing context (recent support interactions, pricing eligibility, competitor comparisons) before finalising high-stakes recommendations
- **Capability 5: Natural Language Churn Explanation** — Synthesises ML feature importances and RAG segment statistics into jargon-free explanations for business users

### Optional Settings

- Probability threshold configurable in `train.py` (default: τ* = 0.52)

## 🛠️ Troubleshooting

| Issue | Solution |
|-------|----------|
| API key not recognised | Verify `GROQ_API_KEY` is valid and set in Streamlit secrets or sidebar |
| Model not found | Run `python train.py` to train and save the model |
| Cold start delay (8–12s) | Expected on Streamlit Community Cloud after inactivity; subsequent interactions are fast |
| Groq rate limit errors | Retry after a short delay; consider upgrading your Groq plan for higher throughput |
| Missing churn drivers | Ensure `data.csv` is present before running `train.py` |

## 📊 Dataset

- **Source:** [Kaggle Telco Customer Churn](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) (IBM Sample Data Sets)
- **Records:** 7,043 customers (7,032 after cleaning)
- **Features:** 20 input features covering demographics, account details, and subscribed services
- **Target:** Binary churn label (Yes / No)
